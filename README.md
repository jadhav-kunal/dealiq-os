# DealIQ OS — Lofty Track

> An AI-native workspace for real estate agents. Built for GlobeHack 2026 at Arizona State University.

DealIQ OS reimagines how real estate agents first experience Lofty as an AI-native platform. Instead of a static widget dashboard, agents get a personalized workspace that adapts to their priorities and automates intelligently based on their trust preferences.

---

## The Problem We're Solving

Lofty has powerful AI capabilities — lead scoring, behavioral signals, Smart Plans — but agents still have to navigate the platform themselves, scan through multiple modules, and mentally prioritize: *"What should I do first?"*

DealIQ OS puts AI front-and-center from the first login.

---

## Key Features

- **Conversational onboarding** — 3-step interview that learns your priorities and AI comfort level
- **Adaptive tab grid** — only your chosen sections appear above the fold, drag to reorder
- **3 trust levels per section** — Auto (AI acts), Ask Me (AI drafts + you approve), Manual (AI suggests)
- **Live AI briefing** — morning summary generated by LLM on every login
- **Why This Matters** — per-item AI reasoning fetched on demand
- **Notification bell** — New Updates moved out of the dashboard into a clean dropdown
- **AI Settings panel** — change trust levels and focus mode at any time

## Getting Started

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/dealiq-os.git
cd dealiq-os
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up environment variables

```bash
cp .env.example .env.local
```

Open `.env.local` and add your API key:

```
VITE_ASU_API_KEY=your_key_here
```

### 4. Start the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) — you'll be taken to onboarding on first load.

### 5. Build for production

```bash
npm run build
npm run preview
```

## How the AI Integration Works

All AI calls go through `src/api/claude.ts`. The Vite dev server proxies requests to avoid CORS and SSL cert issues with the ASU endpoint:

```
Browser → /asu-api/v1/... → Vite proxy (Node, ignores SSL) → https://openai.rc.asu.edu/v1/...
```

Two AI functions are exposed:

- `getMorningBriefing(name, focus, tabs)` — called once on workspace load, generates the daily summary banner
- `getWhyThisMatters(title, data, staticReason)` — called lazily when an agent expands the "Why this matters" section on any item

If the API is unavailable, both functions fall back gracefully to static content.

## The Trust Level System

Each of the 8 workspace sections can be set to one of three AI trust levels:

| Level | Label | Behavior |
|---|---|---|
| `auto` | Auto | AI acts completely — agent sees a confirmation log |
| `approval` | Ask Me | AI drafts the action — agent approves or edits before sending |
| `manual` | Manual | AI suggests what to do — agent composes and executes manually |

Trust levels are set during onboarding and can be changed anytime via the AI Settings panel.

## Design Decisions

**Why tab-based over card-based?**
The original card grid showed everything at once, which recreated the same cognitive load problem Lofty already had. Tabs let agents focus on one area at a time with full context.

**Why a modal instead of a new page?**
Modals keep the agent in context of the full workspace. They can close and immediately see what else needs attention without navigating back.

**Why trust levels instead of a single AI toggle?**
Different parts of the workflow have different stakes. Agents may trust AI to send birthday messages automatically but want to personally review messages to high-value leads. Granular control per section builds trust incrementally.

**Trade-offs made:**
- Mock data used instead of live Lofty API (no access during hackathon)
- AI briefing fetched once on load, not refreshed in real-time
- Drag-and-drop only reorders priority tabs, not sections within a tab panel

**What we'd build next:**
- Real Lofty API integration replacing mock data
- Persistent AI action log ("here's everything AI did today")
- Mobile-optimized layout
- Animated tab transition when trust level changes
- Team view showing AI actions across all agents

## Success Metrics

If this shipped, we'd measure:

- **AI adoption rate** — % of agents who set at least one tab to Auto within 7 days
- **Time-to-first-action** — how quickly agents act on the first recommended item after login
- **Trust level migration** — % of agents upgrading from Manual → Ask Me → Auto over time
- **Daily active usage** — return rate after onboarding (target: 3 days in a row)
- **Action completion rate** — % of AI-drafted messages approved vs dismissed
